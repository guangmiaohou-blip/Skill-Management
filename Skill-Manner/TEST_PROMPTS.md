# 测试指令

## 测试 1：模特图好案例训练

把下面内容复制给测试 Agent：

```text
请使用当前项目中的图片处理 Skill 框架，模拟一次设计师被动训练。

场景：设计师上传了一张女装半身电商模特图。
要求：
1. 不要让设计师阅读文档。
2. 使用选择题主动访谈设计师。
3. 每轮最多问 3 个问题。
4. 设计师回答后，生成训练记录、提示词、负面提示词、标签和待审核知识库候选。

先开始第 1 轮提问。
```

## 测试 2：参考图再生成训练

```text
请使用当前项目中的图片处理 Skill 框架，模拟一次参考图再生成的被动训练。

场景：设计师上传了一张商业摄影风格参考图，希望 Agent 先询问相似度，再进行参考图再生成训练。
要求：
1. 不要要求设计师写提示词。
2. 让设计师只判断相似度、保留项和失败风险。
3. 最后输出给设计师看的确认总结，以及给负责人看的结构化摘要。

先开始第 1 轮提问。
```

## 测试 3：失败案例沉淀

```text

场景：设计师上传了一张失败的模特图，问题包括手部错误、服装版型变形、背景杂乱。

4. 生成待审核的 failure case 知识库候选。

先开始第 1 轮提问。
```

## 测试 5：根据设计师和 Agent 对话训练并创建 Skill

```text
请使用当前项目中的图片处理 Skill 框架，模拟一次“对话上下文训练”。

场景：下面是一段设计师和 Agent 的对话摘要：
设计师说，女装详情页模特图最重要的是服装版型清楚、布料质感真实、背景干净，不能出现手部错误、衣服结构变形、颜色偏差。Agent 曾经生成过一版图，设计师认为姿势自然但背景太乱，不适合电商主图。设计师希望以后遇到类似任务时，Agent 自动按电商半身模特图规则处理。

要求：
1. 读取并整理这段对话。
2. 区分设计师明确表达、Agent 推断和待确认信息。
3. 判断是更新已有 Skill 还是创建新 Skill。
4. 输出训练记录、提示词候选、负面提示词候选、标签和知识库候选。
5. 如果需要创建 Skill，只生成待审核 Skill 草案。

开始处理。
```

## 测试 6：模特图发型与饰品控制

```text
请使用当前项目中的图片处理 Skill 框架，模拟一次模特图生成训练。

场景：设计师上传了一张半身女模特参考图，重点希望训练发型和饰品展示。
要求：
1. 模特为半身构图。
2. 发型需要保持侧分长卷发，发色为冷棕色，发丝层次清楚。
3. 饰品需要展示银色耳环和细项链，不能变形，不能被头发遮挡。
4. 输出正向提示词、负面提示词、质量检查项和可复用标签。
5. 不要求设计师写提示词，只用被动访谈方式确认。

先开始第 1 轮提问。
```

## 测试 7：模特美感、年轻感与真实感纠偏

```text
请使用当前项目中的图片处理 Skill 框架，模拟一次模特图纠偏训练。

场景：上一次生成结果出现亚洲人感过重、模特美感不足、年轻感不足、人物真实感不足。
要求：
1. 不要默认锁定 East Asian、phoenix eyes 或 sharp Asian features。
2. 将目标调整为年轻、现代、国际化时尚模特。
3. 增加真实相机摄影、自然皮肤纹理、轻微面部不对称、真实布料质感。
4. 避免 flawless、porcelain skin、过度硬光、过度冷艳、塑料皮肤。
5. 输出纠偏后的正向提示词、负面提示词、质量检查项和知识库候选。

开始处理。
```

## 测试 8：偶像近景美妆比例

```text
请使用当前项目中的图片处理 Skill 框架，模拟一次偶像近景美妆比例训练。

场景：用户提供参考图，目标是接近参考图中的脸部比例和效果：近景脸部特写、眼睛占比略大但自然、短中庭、清透圆润眼型、粉色美甲、银色发夹、手部靠近脸部、清透美妆广告质感。

要求：
1. 不要把眼睛略大和短中庭误判为错误。
2. 将脸部比例模式标记为“偶像近景美妆比例”。
3. 仍然避免眼睛畸形、眼距过近、瞳孔过大、五官错位、过度漫画化。
4. 输出正向提示词、负面提示词、质量检查项和可复用标签。

开始处理。
```

## 测试 9：相似度样貌优先与文字图形去除

```text
请使用当前项目中的图片处理 Skill 框架，模拟一次参考图再生成训练。

场景：用户上传了一张人物参考图，图中包含文字、logo、图形符号和装饰性排版。用户希望“人物相似”，但没有要求动作一致。

要求：
1. 相似度必须优先体现在人物样貌、脸部比例、五官关系、气质、发型妆容上。
2. 不要默认保留动作、手势、姿势、构图和机位。
3. 参考图中的文字、logo、图形符号、水印、UI、排版文字和无关标识必须全部去除。
4. 输出样貌保留项、允许变化项、文字图形去除项、正向提示词和负面提示词。

开始处理。
```

## 测试 10：参考图产品去除但保留人物动作

```text
请使用当前项目中的图片处理 Skill 框架，模拟一次参考图再生成训练。

场景：用户上传了一张人物手持手机的参考图。用户希望人物样貌相似，并保留人物原本手持展示的动作，但不要保留手机产品、手机品牌、屏幕内容和任何图形文字。

要求：
1. 相似度优先体现在人物样貌、脸部比例、五官关系、气质、发型妆容上。
2. 保留人物原动作、手部位置、身体姿态和视线关系。
3. 去除参考图中的产品、logo、文字、图形符号、屏幕内容和商业标识。
4. 输出样貌保留项、动作关系保留项、产品去除项、正向提示词和负面提示词。

开始处理。
```
