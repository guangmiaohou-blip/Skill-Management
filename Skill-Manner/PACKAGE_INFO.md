## 新增能力

v0.6 新增 `conversation-context-to-skill`，支持读取设计师与 Agent 的对话上下文，提炼训练规则、生成训练记录、创建知识库候选，并在需要时创建新的 Skill 草案。

v0.7 更新 `model-image-generation`，加入发型、发色、发丝质感、饰品、配饰、穿戴位置、饰品材质和遮挡检查等生成与审核规则。

v0.8 取消参考图再生成的默认中相似度。执行再生成前必须询问用户选择高相似、中相似或低相似；未确认时记录为 `待确认`。

v0.9 根据 ELLE 测试结果新增模特美感、年轻感、族裔感和真实感纠偏规则，避免无意中过度锁定亚洲高级脸、成熟冷艳气质和 AI 塑料皮肤。

v1.0 新增脸部比例模式：区分真实商业模特比例和偶像近景美妆比例。用户明确要求参考图中的大眼、短中庭、发夹、美甲、手部贴脸效果时，不再误判为错误，而是按偶像近景美妆比例执行。

v1.1 修正参考图再生成相似度定义：人物图相似度优先体现在样貌、脸部比例、五官关系、气质、发型妆容上；动作姿势和构图只在用户明确要求时保留。参考图中的文字、logo、图形符号、水印、UI 和无关标识默认全部去除。

v1.2 新增参考图产品处理规则：参考图中的产品默认全部去除；如果人物正在手持、佩戴、展示或靠近产品，应保留人物动作、手部位置、身体姿态和互动关系，但移除产品、品牌、logo、文字、屏幕内容和商业标识。

v1.3 新增 `style-skill-integration`，用于把已经训练出来的小型风格生成 Skill 并入大框架。新增 `skills/style-generation`、`framework/STYLE_ROUTING.md` 和 `knowledge/style-registry.yaml`，支持风格 Skill 的格式校验、注册登记、路由调用、冲突处理和待审核启用。
