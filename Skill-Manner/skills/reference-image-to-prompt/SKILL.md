# reference-image-to-prompt

## 用途

当用户上传参考图，并希望逆向转化为提示词、拆解画面、提取风格或生成可复用标签时，使用本 skill。

## 输入

```yaml
image: 用户上传图片
user_request: 用户原始需求
language: 中文
```

## 输出

必须输出：

1. 图片类型。
2. 自动分类结果。
3. 分维度图片分析。
4. 正向提示词。
5. 负面提示词。
6. 推荐参数。
7. 可复用标签。
8. 可写入 records 的结构化记录。

## 分析维度

按以下维度分析：

| 维度 | 要点 |
| --- | --- |
| 主体识别 | 人物、产品、场景、核心物体 |
| 人物或产品细节 | 五官、发型、服装、材质、颜色、品牌元素、产品结构 |
| 姿势动作 | 站姿、坐姿、手势、视线、动作状态 |
| 构图视角 | 景别、角度、主体位置、留白、透视 |
| 光线 | 光源方向、软硬、明暗、阴影 |
| 背景 | 场景、道具、空间、环境风格 |
| 材质 | 皮肤、布料、金属、玻璃、塑料、皮革等 |
| 色彩 | 主色、辅色、饱和度、色温、对比 |
| 风格 | 摄影风格、商业风格、写实程度、艺术倾向 |
| 镜头语言 | 焦段感、景深、运动感、快门感 |
| 后期质感 | 精修程度、锐度、颗粒、胶片感、商业质感 |
| 负面提示词 | 需要避免的画面缺陷 |
| 可复用标签 | 可用于知识库检索的短标签 |

## 提示词结构

使用以下格式：

```text
正向提示词：
主体 + 关键细节 + 姿势动作 + 构图视角 + 光线 + 背景 + 材质 + 色彩 + 风格 + 镜头语言 + 后期质感

负面提示词：
画面错误 + 人体错误 + 产品错误 + 材质错误 + 风格偏差 + 低质量问题
```

## 推荐参数

如果用户没有指定平台或模型，参数字段写 `待定`，但仍给出通用建议：

```yaml
platform: 待定
model: 待定
aspect_ratio: 根据参考图判断
similarity_level: 待确认
style_strength: 中
detail_strength: 中高
```

## 知识沉淀

执行结束后，把结果写入 `records/templates/image-analysis-record.yaml` 的字段结构。成功的提示词可沉淀到 `knowledge/prompt-patterns/`，失败或风险点沉淀到 `knowledge/failure-cases.md`。
