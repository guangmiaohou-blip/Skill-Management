# 设计师被动训练界面流程

## 目标

如果未来要做网页或内部工具，界面只保留设计师必须操作的内容。

## 页面 1：上传图片

设计师看到：

```text
上传一张你想训练 Agent 的图片
```

按钮：

```text
上传图片
```

可选输入：

```text
这张图准备用来做什么？可以不填
```

## 页面 2：快速选择

设计师看到 3 个问题：

```text
这张图是什么类型？
模特图 / 人像图 / 产品图 / 场景图 / 风格参考图 / 不确定

这张图是？
好案例 / 一般 / 失败案例

你希望训练什么？
逆向提示词 / 参考图再生成 / 模特图生成 / 不确定
```

## 页面 3：点选亮点或问题

如果是好案例，显示：

```text
它哪里值得学习？
构图 / 光线 / 姿势 / 模特气质 / 产品展示 / 背景 / 色彩 / 材质 / 商业质感
```

如果是失败案例，显示：

```text
它主要失败在哪里？
人体错误 / 五官问题 / 产品变形 / 服装版型错误 / 材质不真实 / 光线不好 / 背景杂乱 / 风格跑偏 / 质感不足
```

## 页面 4：Agent 自动总结

设计师看到：

```text
我已经整理成训练规则，请确认：

Agent 应该学习：
...

Agent 必须避免：
...

生成提示词：
...
```

按钮：

```text
确认并保存
修改一句
重新分析
```

## 页面 5：完成

设计师看到：

```text
已完成训练记录。
这条经验已进入待审核知识库。
```

可选按钮：

```text
继续上传下一张
查看本次总结
```

## 设计原则

1. 不出现 skill、YAML、JSON 等技术词。
2. 每屏只做一个动作。
3. 所有输入都允许“不确定”。
4. 设计师永远只确认结果，不负责整理结果。
