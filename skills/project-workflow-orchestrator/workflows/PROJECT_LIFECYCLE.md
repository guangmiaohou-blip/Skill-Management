# 图片项目生命周期

## 目标

将设计师的自然语言任务和背景材料自动转换为可执行、可追溯、可审核、可共享的图片项目。

## 状态机

| 状态 | 自动动作 | 设计师动作 | 进入条件 | 退出条件 |
| --- | --- | --- | --- | --- |
| intake | 创建项目目录和主清单 | 补充必要任务信息 | 收到任务 | 分类完成 |
| classified | 识别图片、意图、Skill 和材料 | 确认相似度等必选项 | 已有输入 | 路由完成 |
| running | 执行提示词和图片生成 | 可上传补充材料 | 任务参数齐全 | 产生输出 |
| awaiting-selection | 整理临时图并生成缩略信息 | 选择成功图片并打标签 | 至少一轮输出 | 已选择成功案例 |
| candidate-review | 生成案例、知识和 Skill 更新候选 | 提交审核 | 有成功案例 | 审核通过或退回 |
| revision | 保存问题并重新执行指定环节 | 修改选择或补充材料 | 审核退回 | 重新提交 |
| approved | 锁定审核内容和版本 | 确认发布 | 审核通过 | 发布包完成 |
| shared | 更新共享目录和总 Skill | 无需重复操作 | 发布包完成 | 项目归档 |
| blocked | 保存错误和上下文 | 选择恢复方式 | 无法继续 | 恢复到指定状态 |

## 自动化边界

Agent 可以自动执行：识别、分类、创建目录、复制材料、读取 MD、读取 Skill、生成提示词、调用图像平台、保存临时输出、生成结构化记录、整理候选、更新审核队列和生成发布包。

Agent 必须等待设计师：相似度档位、明确的保留项、成功图片选择、最终案例标签和审核发布确认。

## 完成定义

一个项目只有同时满足以下条件才算完成：

1. 有 `project-manifest.yaml` 和完整状态记录。
2. 成功图片位于 `05_selected/`，未选图片仍位于 `04_temp_outputs/`。
3. 每张成功图片都能反向找到生成参数、提示词、背景材料和使用的 Skill。
4. `knowledge-candidate.yaml` 已经生成并通过审核。
5. 共享发布包包含案例、Skill 版本、MD 文档、规范和 `release-manifest.yaml`。
