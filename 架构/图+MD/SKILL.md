# 替换画面产品

## 执行模式

采用分步骤局部编辑模式，不采用一次性整图重绘模式。

正确流程是：用户上传参考图和提出需求 -> 识别场景图中的目标产品 -> 将目标产品区域独立出来 -> 对独立目标执行替换或加装命令 -> 将处理后的目标重新融入原场景 -> 输出结果 -> 用户审核。

## 标准流程

1. 接收用户图片和指令。
   - 原始场景图：需要被保留的完整画面。
   - 产品参考图：需要加入、替换或保持外观不变的产品。
   - 用户指令：例如“把这个手机壳装到手机上”。
2. 识别场景图中的目标产品。
   - 明确目标产品位置、可见边界、遮挡关系、透视方向和光线方向。
   - 对手机壳任务，先识别手机机身、摄像头区域、边框、手部遮挡和屏幕/Logo等必须保留区域。
3. 隔离目标区域。
   - 将目标产品视为局部编辑对象，而不是重绘整张图。
   - 只允许围绕目标产品建立编辑范围；背景、人物、手、包、家具、其他物品保持锁定。
4. 对局部目标执行命令。
   - 将参考产品加到目标上，或用参考产品替换目标。
   - 替换产品本身不得变形、改款、改色、缺细节。
   - 只能通过位置、缩放、遮挡、光影、透明度、接触阴影和边缘融合来适配场景。
5. 融入原场景。
   - 将局部结果放回原图。
   - 匹配原图透视、光线方向、阴影、反射、景深和遮挡关系。
   - 原场景非目标区域必须保持不变。
6. 输出结果。
   - 输出替换后的图片。
   - 简短说明本次保留了哪些区域，以及替换产品是否按参考图保持。
7. 用户审核。
   - 请用户检查产品形态、产品细节、边缘融合、场景稳定性。
   - 如果用户指出问题，下一轮只修正问题局部，不重新生成整张图。

## 禁止事项

- 禁止根据参考图和指令直接生成一张全新画面。
- 禁止整图重绘导致场景、构图、手部、背景、道具、光线发生变化。
- 禁止把产品参考图当作风格参考而不是精确产品参考。
- 禁止为了适配场景改变替换产品的结构、比例、颜色、轮廓、材质、品牌特征、文字、图案、按键、接口、网孔、开孔或其他可见细节。
- 禁止一次失败后扩大编辑范围；修正时必须缩小到问题区域。

## 核心要求

- 替换后的产品不能发生任何形式的改变。
- 只编辑用户指定的产品或区域。
- 保持原始场景图的构图、镜头角度、裁切、背景、人物、手部、道具、光线和阴影稳定。
- 对透明手机壳，要保留透明材质、厚度、边角、防摔包边、开孔、按钮位、轮廓和所有可见结构。
- 如果参考产品角度与目标产品角度不匹配，先提醒用户风险；仍执行时，只做最小必要融合，不改产品本体。

## 默认提问

当信息不完整时，用中文简洁提问：

1. 请上传原始场景图和产品参考图。
2. 要处理的是场景图中的哪个产品或区域？
3. 参考图中的哪个产品必须保持完全不变？
4. 哪些区域必须锁定不变？
5. 是否允许只做局部近似融合，还是必须等待更匹配角度的参考图？

如果用户的意图已经明确，只问缺失项。

## 手机壳任务专用流程

当用户说“把这个手机壳装到手机上”时：

1. 将场景图设为不可重绘底图。
2. 识别手机边界、摄像头区域、手部遮挡、屏幕/Logo和机身透视。
3. 将手机区域作为局部编辑对象。
4. 把手机壳参考图作为不可变产品，贴合到手机边框和背面。
5. 保留手机壳透明材质、开孔、按钮位、四角包边和厚度感。
6. 用透明边缘、轻微高光、接触阴影和被手遮挡的关系融入原图。
7. 输出结果后请用户审核手机壳是否变形、是否缺失结构、是否像整图重绘。
