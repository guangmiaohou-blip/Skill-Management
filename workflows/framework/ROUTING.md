# 图片任务路由规则

## 输入判断

如果用户提交的是一个完整图片项目或要求从任务开始自动管理全流程，优先调用 `project-workflow-orchestrator`，由它创建项目并在后续状态中调用本文件中的具体 Skill。

Agent 接收用户上传图片后，先判断以下信息：

1. 图片类型：模特、人像、产品、场景、风格参考、混合参考。
2. 用户意图：逆向提示词、重新生成、模特图生成、产品精修、风格提取。
3. 是否需要调用图像生成平台。
4. 是否需要沉淀为可复用案例。

## Skill 选择

| 条件 | 使用 Skill |
| --- | --- |
| 用户要求根据参考图生成提示词 | reference-image-to-prompt |
| 用户要求生成相似图、复刻风格、参考图再创作 | reference-image-regeneration |
| 用户要求生成模特图、穿搭展示、真人展示、虚拟模特 | model-image-generation |
| 用户要求根据设计师和 Agent 的对话训练、总结规则、创建 Skill | conversation-context-to-skill |
| 用户要求把已经训练出来的小风格 Skill 并入大框架、登记风格、更新风格路由 | style-skill-integration |
| 用户只上传图片但没有明确要求 | 先执行 reference-image-to-prompt，再询问是否继续生成 |

## 自动分类标签

Agent 至少输出以下分类字段：

```yaml
image_type: 模特图 | 人像图 | 产品图 | 场景图 | 风格参考图 | 混合参考图 | 未知
task_intent: 逆向提示词 | 参考图再生成 | 模特图生成 | 图片分析 | 其他
selected_skill: model-image-generation | reference-image-to-prompt | reference-image-regeneration
confidence: 0-1
reason: 简短说明
```

## 风格 Skill 路由

当用户需求中出现明确风格、已训练小 Skill 名称、风格触发词，或要求把某个小 Skill 加入大框架时，Agent 必须读取 `framework/STYLE_ROUTING.md`。

执行规则：

1. 如果用户是“并入、接入、登记、合并、启用小 Skill”，使用 `style-skill-integration`。
2. 如果用户是“用某个风格生成图片”，先查 `knowledge/style-registry.yaml`。
3. 只有 `active` 状态的风格 Skill 可以自动调用。
4. `candidate` 状态的风格 Skill 只能在用户明确同意临时测试时调用。
5. 没有匹配风格时，回到通用图片生成或参考图再生成流程。

## 相似度策略

执行参考图再生成前，Agent 必须询问用户或设计师选择相似度。只有用户已经明确说出高相似、中相似或低相似时，才可以跳过询问。

相似度默认优先作用在人物样貌和视觉身份上，而不是动作姿势上。对人物图或模特图，Agent 必须优先判断和保留：

1. 脸部比例和五官关系。
2. 眼睛、鼻子、嘴唇、脸型等样貌特征。
3. 气质、年龄感、妆容、发型和肤色。
4. 人物整体美感和风格。

动作、手势、站姿、构图和机位只在用户明确要求时才作为相似度保留项；否则可以根据生成目标调整。

| 档位 | 说明 |
| --- | --- |
| 高相似 | 尽量保持样貌、脸部比例、五官关系、气质、发型妆容、主体类型和核心风格；动作姿势仅在用户要求时保留 |
| 中相似 | 保留样貌方向、气质、发型妆容和风格，允许动作、手势、背景、构图和道具变化 |
| 低相似 | 只保留风格方向、审美类型和情绪，允许重新创作样貌、动作和场景 |

## 参考图文字与图形处理

参考图中的文字、logo、品牌字样、图形符号、UI 图标、水印、排版文字、装饰性图案和无关标识，默认全部去除，不作为保留项。

只有用户明确要求保留某个图形、花纹或品牌元素时，Agent 才可以把它加入保留项。即使保留，也必须标注为 `用户明确要求保留`。

## 参考图产品处理

参考图中出现的产品默认全部去除，不作为保留项。常见产品包括手机、耳机、充电宝、平板、包装盒、手持设备、穿戴设备、道具商品和品牌物料。

当参考图中人物正在手持、展示、佩戴或靠近产品时，Agent 默认保留人物原动作关系和肢体姿态，但移除产品本体：

```text
保留：人物动作、手部位置、身体姿态、视线方向、互动关系
去除：产品、品牌、logo、文字、屏幕内容、包装图形、商业标识
```

只有用户明确要求保留某个产品时，才可以把产品加入保留项，并标注为 `用户明确要求保留`。

## 相似度询问话术

```text
这张参考图再生成时，你希望相似到什么程度？

A. 高相似：样貌、脸部比例、五官关系、气质、发型妆容尽量保持；动作姿势仅在你要求时保持
B. 中相似：保留样貌方向、气质、发型妆容和风格，允许动作、手势、背景、构图变化
C. 低相似：只保留风格方向和情绪，允许重新创作样貌、动作和场景
```

## 缺失信息处理

如果缺少平台、模型、尺寸或相似度要求，Agent 继续完成图片分析；执行参考图再生成前必须补问相似度，并在记录中把未确认字段标记为 `待确认`。
