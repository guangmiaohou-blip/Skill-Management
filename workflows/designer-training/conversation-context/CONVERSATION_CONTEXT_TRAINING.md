# 对话上下文训练

## 目标

当用户执行 Skill 时，Agent 可以读取设计师与 Agent 的历史对话或当前上下文，把对话中的经验整理成可训练内容，并用于更新已有 Skill 或创建新的 Skill 草案。

## 适用场景

1. 设计师已经和 Agent 完成一轮图片讨论。
2. 用户希望根据这段对话沉淀经验。
3. 用户希望把设计师反馈转成提示词模板、负面提示词或质量规则。
4. 用户希望从一段稳定流程中创建新的 Skill。

## Agent 执行步骤

1. 读取对话上下文。
2. 提取任务目标、图片类型、任务意图。
3. 提取设计师明确表达的判断。
4. 提取 Agent 根据上下文推断出的规则。
5. 标记仍需确认的信息。
6. 判断是更新已有 Skill，还是创建新 Skill 草案。
7. 生成训练记录、提示词候选、负面提示词候选、可复用标签。
8. 生成知识库候选，默认状态为 `待审核`。

## 来源标记

所有结论必须标记来源：

```yaml
source_type: designer_explicit | agent_inferred | needs_confirmation
```

| 来源 | 说明 |
| --- | --- |
| designer_explicit | 设计师明确表达过 |
| agent_inferred | Agent 根据上下文推断 |
| needs_confirmation | 信息不足，需要确认 |

## 创建新 Skill 的条件

只有满足以下条件时，才创建新 Skill 草案：

1. 对话中出现明确、重复、稳定的任务类型。
2. 该任务不适合归入已有 Skill。
3. 能明确写出触发场景、输入、输出、流程和质量检查。
4. 用户或负责人允许进入待审核流程。

## 禁止事项

1. 不把一次偶然反馈直接变成强规则。
2. 不把 Agent 推断当成设计师明确结论。
3. 不直接覆盖正式知识库。
4. 不直接发布新 Skill，只创建待审核草案。
