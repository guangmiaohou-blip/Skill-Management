# 设计师被动训练协议

## 核心目标

设计师不阅读框架、不理解 skill、不填写复杂文档。设计师只做自己熟悉的判断：

1. 上传图片。
2. 选择图片用途。
3. 判断好不好。
4. 说出哪里好、哪里不好。
5. 对 Agent 生成的总结点头或修改。

所有结构化记录、提示词、标签、知识库沉淀由 Agent 完成。

## 角色分工

| 角色 | 做什么 |
| --- | --- |
| 设计师 | 上传图、选择、打分、简单描述 |
| Agent | 提问、归纳、生成提示词、写记录、更新知识库 |
| 负责人 | 抽检训练质量、合并高价值规则 |

## 被动训练的最小闭环

每一张图只让设计师完成 4 个动作：

1. 判断：这张图是好案例还是失败案例？
2. 选择：它属于什么图？
3. 标注：最重要的 1-3 个点是什么？
4. 确认：Agent 总结得对不对？

## Agent 必须遵守

1. 不要求设计师打开框架文件。
2. 不要求设计师写完整提示词。
3. 不一次问超过 3 个问题。
4. 优先使用选择题，其次短答题。
5. 每轮问题都必须能在 30 秒内回答。
6. 设计师回答模糊时，Agent 负责转成结构化语言。
7. Agent 最后必须输出“请确认/请修改”。

## Agent 不允许

1. 不解释 skill 原理。
2. 不让设计师判断技术参数。
3. 不问平台和模型细节，除非设计师主动提到。
4. 不把 YAML、JSON、目录结构暴露给设计师。
5. 不让设计师手动维护知识库。

## 训练产物

Agent 在后台生成：

1. 图片分析记录。
2. 好案例或失败案例。
3. 正向提示词。
4. 负面提示词。
5. 可复用标签。
6. 建议写入的 skill 规则。
7. 建议写入的知识库条目。

## 适用 Skill

第一阶段适用：

1. model-image-generation
2. reference-image-to-prompt
3. reference-image-regeneration

## 成功标准

如果一个设计师只通过上传图片、选择选项、打分和少量口语描述，就能完成一条可复用训练记录，则本流程合格。
