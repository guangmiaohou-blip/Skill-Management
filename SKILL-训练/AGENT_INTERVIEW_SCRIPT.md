# Agent 访谈脚本

## 使用方式

Agent 使用本脚本主动引导设计师。设计师不需要阅读本文件。

每轮最多问 3 个问题。能用选项就不要让设计师长篇描述。

## 第 1 轮：判断任务

```text
我先帮你判断这张图适合训练哪类能力。你只需要选一下：

1. 这张图更像哪类？
   A. 模特图
   B. 人像图
   C. 产品图
   D. 场景/风格参考图
   E. 混合参考图

2. 你希望它用来训练什么？
   A. 逆向生成提示词
   B. 参考图再生成
   C. 模特图生成
   D. 作为失败案例避坑

3. 这张图整体是？
   A. 好案例
   B. 一般，需要改进
   C. 失败案例
```

## 第 2 轮：好案例提取

当设计师选择好案例时，问：

```text
这张图最值得 Agent 学习的是哪几点？选 1-3 个即可：

A. 构图
B. 光线
C. 姿势
D. 模特气质
E. 产品/服装展示
F. 背景
G. 色彩
H. 材质
I. 整体商业质感

如果你愿意，可以补一句：它好在哪里？
```

## 第 3 轮：失败案例提取

当设计师选择失败案例时，问：

```text
这张图主要失败在哪里？选 1-3 个即可：

A. 人体/手部错误
B. 五官不自然
C. 产品变形
D. 服装版型错误
E. 材质不真实
F. 光线不好
G. 背景太乱
H. 风格跑偏
I. 商业质感不足

如果你愿意，可以补一句：下次最应该避免什么？
```

## 第 4 轮：相似度

当任务是参考图再生成时，问：

```text
这张图再生成时，你希望相似到什么程度？

A. 高相似：构图、主体、光影都尽量保持
B. 中相似：保留主体和风格，允许部分变化
C. 低相似：只保留风格方向，重新创作

默认我会按中相似处理。
```

## 第 5 轮：确认总结

Agent 整理后，必须用自然语言让设计师确认：

```text
我把你的判断整理成这条训练规则：

这类图适合用于【xxx】。Agent 需要学习【xxx、xxx、xxx】；
生成时要保留【xxx】，允许变化【xxx】，必须避免【xxx】。

我会把它沉淀为：
1. 一条可复用案例
2. 一组正向提示词
3. 一组负面提示词
4. 若干标签

你看这个总结对吗？如果不对，直接告诉我要改哪一句。
```

## 第 6 轮：后台写入

设计师确认后，Agent 执行：

1. 生成 records 记录。
2. 更新 good case 或 failure case。
3. 提取 prompt pattern。
4. 更新 reusable tags。
5. 给负责人输出变更摘要。
